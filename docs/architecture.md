## Каскадная классификация

Система использует каскадный подход — сначала применяются лёгкие и быстрые методы,
затем при необходимости подключаются более тяжёлые:

```
Документ
   │
   ▼
┌─────────────────────┐
│ Уровень 0: Сборщик  │  Мониторинг почты / папки
└──────────┬──────────┘
           ▼
┌─────────────────────┐
│ Уровень 1: Правила  │  Regex + ключевые слова (<1 сек)
│ (Вариант A)         │
└──────────┬──────────┘
           │ confidence < 0.7
           ▼
┌─────────────────────┐
│ Уровень 2: ML       │  TF-IDF + SVM (1-2 сек)
│ (Вариант B)         │
└──────────┬──────────┘
           │ confidence < 0.7
           ▼
┌─────────────────────┐
│ Уровень 3: LLM      │  Локальная LLM через Ollama (5-30 сек)
│ (Вариант D)         │
└──────────┬──────────┘
           │ confidence < 0.5
           ▼
┌─────────────────────┐
│ Ручная классификация│  Запрос к пользователю через UI
└─────────────────────┘
```

## Варианты классификаторов

### Вариант A: Правила + OCR
- **Сложность**: 2/5
- **Точность**: 70-85%
- **RAM**: 1-2 GB
- **Принцип**: Regex-паттерны, ключевые слова, анализ имени файла
- **Когда использовать**: Стандартные документы с чёткой структурой

### Вариант B: ML (TF-IDF + SVM)
- **Сложность**: 3/5
- **Точность**: 85-92%
- **RAM**: 2-4 GB
- **Принцип**: Извлечение TF-IDF признаков → классификация SVM/Random Forest
- **Когда использовать**: Документы с вариативной структурой

### Вариант C: Нейросеть (MobileNet + SBERT)
- **Сложность**: 4/5
- **Точность**: 92-97%
- **RAM**: 4-8 GB
- **Принцип**: Мультимодальный анализ (визуальный + текстовый + метаданные)
- **Когда использовать**: Сложные документы, плохое качество сканов

### Вариант D: LLM (Ollama)
- **Сложность**: 5/5
- **Точность**: 95-99%
- **RAM**: 16+ GB
- **Принцип**: Анализ текста локальной языковой моделью
- **Когда использовать**: Нестандартные или неизвестные типы документов

## Типы документов

| Тип | Описание | Ключевые поля |
|-----|----------|---------------|
| Счёт-фактура | Счёт на оплату | ИНН, КПП, Продавец, Покупатель |
| Акт | Акт выполненных работ | Исполнитель, Заказчик |
| Договор | Договор между сторонами | Предмет договора, Реквизиты |
| ТОРГ-12 | Товарная накладная | Грузоотправитель, Грузополучатель |
| Платёжное поручение | Банковский перевод | БИК, Банк получателя |

## Обратная связь и самообучение

1. Система классифицирует документ
2. Если уверенность высокая (>85%) — автоматическая сортировка
3. Если уверенность низкая — запрос к пользователю
4. Подтверждения/исправления сохраняются в SQLite
5. После накопления N исправлений — автоматическое переобучение ML-модели
